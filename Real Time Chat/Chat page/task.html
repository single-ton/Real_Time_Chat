Description

In the stage, you'll create a chat application by implementing the page that has three main components:

  • A text field to input messages;
  • A button that will send an input message;
  • A message container where messages will appear.

The stage focuses on the page design, use HTML/CSS to implement it.

To finish the project, you may need to search for some information on your own. Revisit topics, refer to documentation, or search for some additional guides and tutorials if you need to.

When you run tests for the first time, they will install a browser that will be used to test the program. Make sure the IDE, the node, and the browser are not blocked by a firewall; otherwise, tests can fail.
Objectives
  • Create the static folder in the resources folder. In the static folder, create index.html file. It will be the chat page. It is a good practice to store CSS and JS code separately, you can create two (or more) files to store CSS/JS in the same folder. These additional files can have any name.

If everything is correct, after running the program and navigating to http://localhost:28852 in a browser, you should see the content of the index.html file.

  • The page should have a title tag with the text Chat.
  • Add an input field that will be used to input messages. You can use the textarea or input tag. The tag should have an ID of input-msg;
  • Add a button with the ID of send-msg-btn that will be used to send messages. The button can have any name or contain an image;
  • Add a tag with the ID of messages. This element will contain all received messages. In this stage, it will contain only hard-coded messages;
  • The element with the ID of messages should contain five elements with the class message. Each of these inner elements should have some text (a message). For example, <div id="messages"><div class="message">Hi</div>...</div>.

The order of mentioned elements is not relevant. Furthermore, it is not mandatory to place the mentioned elements directly in the body tag and there can be some intermediate element(s). The same applies to the message container, which can contain some intermediate elements with elements (messages) with the message class.

  • Add the CSS styles. The main goal in this stage is to set the size of elements and position them. Tests will not check CSS in this stage.
