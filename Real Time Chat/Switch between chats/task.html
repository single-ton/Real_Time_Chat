Description

In this stage, continue to implement the private chat — allow users to switch between chats. A click on a user in the user list should open a chat with that user. Group chat will remain, but it will be hidden under a button.

Moreover, to make it clear, what chat is opened — public or private, add a new element that will show the currently opened chat. The element will contain the text Public chat if a public chat is opened or a username of a user with which a private chat was opened.

Theory

The article Remove all child elements of a DOM node in JavaScript on Stack Overflow shows how to clear a container with elements using JS.

Objectives
  • Make users in the user list clickable. When a user in the user container is clicked, the component with an messages ID should become empty (all messages should be removed);
  • Add a new button with the public-chat-btn ID. If the current chat is private, the element with the messages ID should be cleared and refilled with public messages after the button is clicked. If the current chat is public, the button should do nothing;
  • The page should have a new tag with the chat-with ID that indicates which chat is opened. The component text should change when a user opens a private or a public chat. When public chat is opened, the component text should be Public chat. And If private chat is opened, the element should contain the name of the user with which private chat is opened.
  • After a login, open the public chat by default. The chat-with tag should contain Public chat and a click on send button sends a public message;
  • When a public message is received, it should only be added to the message container when the public chat is opened. For example, If user A opens a chat with user B, and user B sends a public message, this public message should not appear in the opened chat of A and B, and should only appear in all opened public chats;
  • Optionally, add some styles. For example, the hover effect for users in the user list.
